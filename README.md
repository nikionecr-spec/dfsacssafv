# Aura — Premium Flutter Productivity App

Aura is a fast, beautiful, offline-first productivity app for Android that blends
**Notes + Calendar + Tasks + Focus timer + Developer tools** into one cohesive,
premium experience. It is built with Flutter, Dart, Riverpod and SQLite, using a
clean, feature-first architecture.

> **Design goals:** buttery-smooth (120 Hz where supported), low memory/CPU,
> small APK, and a bespoke Material 3 look with *controlled* glassmorphism —
> beauty that never costs frames.

---

## ✨ Features

| Module | Highlights |
| --- | --- |
| **Dashboard** | Greeting + dual (Jalali/Gregorian) date, today's plan, recent notes, live focus session, productivity stats, quick actions. |
| **Notes** | Full markdown, live preview, **in-editor syntax highlighting**, tags, folders, pin, fast debounced search, link-to-date, undo/redo, text zoom, color. |
| **Calendar** | Jalali **and** Gregorian, month/week/day views, ultra-smooth month paging, events with colors, links to notes & tasks, dual-date display. |
| **Tasks** | Priorities, deadlines with reminders, checklists with progress, swipe-to-delete, daily plan. |
| **Pomodoro** | Configurable focus/break, auto phase cycling, immersive Focus Mode, custom-painted animated ring, weekly stats chart, ongoing notification with controls. |
| **Notifications** | Task/event reminders, ongoing Pomodoro notification with Pause/Skip/Stop actions. |
| **Developer tools** | Base conversion (bin/oct/dec/hex), bitwise (AND/OR/XOR/NOT/shifts), ASCII table, timestamp converter, byte converter. |

## 🏗 Architecture

Feature-first **clean architecture**. Each feature owns its layers:

```
lib/
├── main.dart                 # bootstrap: DB, notifications, high-refresh, DI
├── app.dart                  # MaterialApp + theming
├── core/
│   ├── theme/                # colors, dimens, Material 3 themes
│   ├── database/             # sqflite schema + connection
│   ├── notifications/        # flutter_local_notifications wrapper
│   ├── settings/             # settings model, repository, controller
│   ├── providers/            # DI provider seams (overridden in main)
│   ├── router/               # bottom-nav shell
│   ├── utils/                # dates (Jalali), haptics, debouncer, ids
│   └── widgets/              # GlassCard, AuroraBackground, AuraScaffold, …
└── features/
    ├── dashboard/
    ├── notes/            {domain, data, presentation}
    ├── calendar/         {domain, data, presentation}
    ├── tasks/            {domain, data, presentation}
    ├── pomodoro/         {domain, data, presentation}
    ├── developer_calculator/ {domain, presentation}
    └── settings/         {presentation}
```

- **State management:** Riverpod (no code-gen). Repositories expose `Future`s;
  `FutureProvider`s read them and are invalidated after writes.
- **Persistence:** raw `sqflite` with hand-tuned indexes — no build_runner step.
- **Dates:** [`shamsi_date`](https://pub.dev/packages/shamsi_date) for reliable
  Jalali ↔ Gregorian conversion.

### Performance choices
- `const` widgets everywhere; `RepaintBoundary` around cards and the backdrop.
- Backdrop blur is **opt-in** (`GlassCard(blur: …)`) — used only on the nav bar
  and bottom sheets, never in scrolling lists.
- The animated background uses wide radial gradients (no blur) and is muted
  automatically when its tab is off-screen (`TickerMode`).
- `IndexedStack` keeps tab state; selective rebuilds via `ref.watch(...select())`.
- Custom-painted timer ring & stats chart instead of a charting dependency.
- `flutter_displaymode` opts into 90/120 Hz on supported devices.
- Release build enables R8 + resource shrinking + Dart tree-shaking.

## 🚀 Getting started

**Prerequisites:** Flutter **3.27+** (stable), Android SDK, a device/emulator.

```bash
flutter pub get

# If the platform folders are missing on your machine, generate the gradle
# wrapper + glue once (this repo already ships the Android config):
#   flutter create --platforms=android --org com.nikione .

flutter run                     # debug
flutter build apk --release     # release APK (optimized)
flutter build appbundle         # Play Store bundle
```

Run the tests and static analysis:

```bash
flutter analyze
flutter test
```

> **Note on this repository:** it was generated in a sandbox without network
> access to the Flutter SDK / pub.dev, so `flutter pub get`, `analyze`, `test`
> and the APK build must be run on your machine. The Android `gradle-wrapper.jar`
> is intentionally git-ignored (Flutter convention) and is regenerated by the
> Flutter tool on first build.

## 🔔 Notifications setup

Permissions and receivers are pre-wired in `AndroidManifest.xml`, and
core-library desugaring is enabled in `app/build.gradle` (required by
`flutter_local_notifications`). On first run, open **Settings → Enable reminders**
to grant notification + exact-alarm permissions.
